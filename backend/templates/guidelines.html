Deployment Guidelines

Important deployment prerequisites, operational recommendations, security checks and best practices before executing Terraform templates and infrastructure automation.

🟢 DO's

  • ✔ Configure AWS CLI properly using valid credentials.
  • ✔ Keep Terraform installed and initialized before deployment.
  • ✔ Ensure Docker and Kubernetes are properly configured.
  • ✔ Maintain valid AWS IAM permissions.
  • ✔ Keep ports 22, 80 and 443 open where required.
  • ✔ Verify VPC, Subnets, Route Tables and Internet Gateway.
  • ✔ Push complete and tested source code to GitHub.
  • ✔ Use only validated Terraform templates.
  • ✔ Monitor deployment logs continuously during execution.
  • ✔ Maintain proper naming conventions for infrastructure resources.
  • ✔ Keep sufficient AWS Free Tier quota and billing checks enabled.
  • ✔ Store Terraform state files securely using remote state management.

🔴 DON'Ts

  • ❌ Do not delete the default VPC without replacement configuration.
  • ❌ Do not remove Internet Gateway from public subnets.
  • ❌ Do not expose AWS Secret Keys publicly.
  • ❌ Do not block ports 22, 80 or 443 unnecessarily.
  • ❌ Do not upload incomplete or corrupted Terraform templates.
  • ❌ Do not disable IAM permissions required for deployment.
  • ❌ Do not modify terraform.tfstate manually.
  • ❌ Do not terminate infrastructure resources during deployment.
  • ❌ Do not hardcode sensitive credentials inside Terraform files.
  • ❌ Do not exceed AWS Free Tier limits unknowingly.
  • ❌ Do not ignore Terraform plan output before apply operation.
  • ❌ Do not deploy applications without monitoring and logging enabled.
